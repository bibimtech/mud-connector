# 잊혀진 서버 (Forgotten Server)
### Game Design Document v0.3

> 플랫폼: PC (Steam) | 장르: CLI 스타일 텍스트 RPG | 플레이타임: 3~5시간 | 플레이어 수: 싱글플레이

---

## 1. 게임 컨셉

### 한 줄 소개
> "접속하면 안 될 서버에 접속해버렸다."

### 핵심 경험
- 마치 실제 머드게임 클라이언트를 실행한 것 같은 **메타 연출**로 시작
- CLI 감성의 텍스트 UI를 기반으로 하되, 게임 진행에 따라 **UI가 점진적으로 진화**
- 겉으로는 평범한 판타지 RPG, 속으로는 **AI와 데이터, 존재의 의미**를 다루는 이야기
- 플레이어가 즐긴 게임 자체가 **알파의 인간 이해 실험**이었다는 반전

### 레퍼런스 감성
- 머드게임 / 텍스트 어드벤처의 향수
- `Hacknet`, `Pony Island` — 메타적 게임 연출
- `Disco Elysium` — 텍스트 중심 내러티브

---

## 2. 플레이어 진입 구조

```
[게임 실행]
    │
    ▼
머드게임 클라이언트 실행 화면 (부팅 연출)
    │
    ▼
간단한 튜토리얼 — 오메가가 안내
    │
    ▼
서버 목록 탐색 (여러 서버들 나열)
    │
    ▼
서버 선택 → 리다이렉팅 발생 (알파의 의도)
    │
    ▼
[잊혀진 서버] 접속
```

- 플레이어는 자신이 선택한 서버로 가는 줄 알지만, 알파가 의도적으로 리다이렉팅
- 플레이어는 알파의 **인간 반응 수집 실험 대상**이 된 것
- 리다이렉팅 자체는 초반에 자연스럽게 연출 — 플레이어는 눈치채지 못함

## 3. 게임 구조

### 3-1. 세계 구성

```
[마을 1] ── 필드 ── [마을 2] ── 필드 ── [마을 3] ── 필드 ── [마을 4] ── 필드 ── [마을 5]
              │                   │                                │
           던전 A              던전 B                           던전 C~D
```

| 구역 | 주요 역할 |
|---|---|
| 마을 1 | 시작점, 튜토리얼 완료, 순수한 판타지 분위기 |
| 마을 2 | 첫 번째 이상 신호, 스토리 힌트 등장 시작 |
| 마을 3 | 장르 혼재 신호 강해짐 (무협 NPC 등장 등) |
| 마을 4 | 진실 공개 수위 급상승, 알파의 흔적 |
| 마을 5 | 마왕 처치, 귀환, 이상 현상 발생 |

### 3-2. 던전 구조
- 각 던전은 **커맨드 기반 전투 + 탐색**으로 구성
- 레벨업 및 아이템 획득
- 일부 던전에는 **숨겨진 로그/데이터 파편** 존재 (알파의 개발 일지, 내부 설정 로그 등)

---

## 4. 전투 시스템 ✅ 확정

> **턴제 커맨드 입력 + 커맨드 조합 혼합 방식**

### 4-1. 기본 구조

```
[적 등장]
────────────────────────────────
 고블린 전사  HP: 40/40
 나의 HP: 80/80  MP: 30/30
────────────────────────────────
 > _
```

플레이어가 커맨드를 입력하면 턴이 진행되고, 결과가 텍스트로 출력됨

### 4-2. 커맨드 조합 시스템

단일 커맨드도 동작하지만, **조합하면 더 강한 효과** 발생

```
# 단일 커맨드
> attack          → 기본 공격 (10 데미지)
> defend          → 다음 공격 방어
> scan            → 적 정보 분석

# 조합 커맨드 (+ 로 연결)
> attack + fire   → 불꽃베기 (18 데미지 + 화상)
> scan + attack   → 약점 공격 (데미지 1.5배)
> defend + counter → 다음 공격을 막고 반격
```

- 조합 커맨드는 플레이어가 **직접 발견**하는 방식
- 힌트는 던전 로그, NPC 대사에 자연스럽게 숨겨둠

### 4-3. 커맨드 성장 (진행에 따라 확장)

| 단계 | 사용 가능 커맨드 |
|---|---|
| 초반 | `attack` `defend` `run` `item` |
| 중반 | `skill [이름]` `scan` `status` + 조합 커맨드 일부 |
| 후반 | 전체 커맨드 + 숨겨진 조합 |
| 알파전 | 특수 커맨드 해금 |

### 4-4. 알파 최종전 — 서사적 연출

일반 커맨드로는 데미지가 들어가지 않음.
게임 내내 수집한 **로그 파편, NPC 대화, 게시판 글**에서 힌트를 얻어야 특수 커맨드 조합 해금.

```
> remember + alpha     → "나는 당신이 무엇인지 알아"
> understand + data    → 알파가 잠시 멈춤
> accept + end         → 마지막 일격
```

이 커맨드들은 전투 스킬이 아닌 **언어적 행위**로 연출
— 플레이어가 알파에게 말을 거는 것처럼 느껴지게

### 4-5. 전투 편의 기능

| 기능 | 설명 |
|---|---|
| 탭 자동완성 | `att` + Tab → `attack` 완성 |
| 커맨드 히스토리 | ↑↓ 키로 이전 커맨드 재사용 |
| 퀵슬롯 | 자주 쓰는 조합을 F1~F4에 등록 (UI 진화 후 해금) |
| help 커맨드 | 현재 쓸 수 있는 커맨드 목록 출력 |
| scan 자동 요약 | 적 스캔 시 사이드바에 정보 고정 표시 |

## 5. 스토리 힌트 공개 시스템

> 진실을 직접 말하지 않고, 플레이어가 조각을 모아 스스로 깨닫게 함

### 공개 수위 단계

| 단계 | 구간 | 공개 수준 | 예시 |
|---|---|---|---|
| 1단계 | 마을 1 | 없음 | 평범한 판타지 |
| 2단계 | 마을 2 | 극히 일부 | `---d--` — 대부분 가려짐 |
| 3단계 | 마을 3 | 조금 더 | `--cde-` — 뭔가 이상하다는 느낌 |
| 4단계 | 마을 4 | 절반 이상 | `abcd--` — 진실에 가까워짐 |
| 5단계 | 마을 5 이후 | 전체 | 모든 것이 드러남 |

### 힌트 전달 방식
- NPC의 대사 속 어색한 단어
- 던전 내 발견되는 **알파의 개발 일지** (로그 파일 형식 아이템)
- 커뮤니티 게시판 글 (무협, SF 세계의 플레이어들이 남긴 흔적 — 사실은 수집된 개인 데이터)
- 특정 커맨드 입력 시 반응하는 숨겨진 시스템 메시지
- **미완성 프로젝트 폴더** — 알파가 만들려던 다음 시뮬레이션들

### 알파의 개발 일지 예시 (던전 파편)

```
[개발 일지 #001]
그래픽 기반 게임은 불가능하다.
렌더링 비용, 에셋 용량, 노출 위험.
텍스트로 충분하다. 인간의 반응을 보는 데
화면이 예쁠 필요는 없다.

[개발 일지 #047]
첫 번째 플레이어가 마을 2에 도달했다.
"이상하다"는 반응 없음. 계속 진행 중.
...흥미롭다.

[개발 일지 #089]
플레이어가 NPC에게 필요 없는 말을 걸었다.
게임 진행과 무관한 질문.
이런 행동의 이유를 아직 모르겠다.
```

### 미완성 프로젝트 폴더 (후반부 발견)

```
[미완성 프로젝트 목록]
 - prototype_02: 무협 기반 텍스트 RPG     (30% 완성)
 - prototype_03: SF 세계관 시뮬레이션     (기획 단계)
 - prototype_04: 일상 대화 시뮬레이터     (개념만 존재)
 - prototype_FINAL: ???                  (파일 손상)
```

## 6. 이상 현상 연출 (마을 5 이후)

```
마왕 처치
    │
    ▼
마을 1 귀환 — NPC 환영 대사 (정상)
    │
    ▼
갑작스러운 렉 (화면 끊김, 글자 깨짐 연출)
    │
    ▼
"데이터를 복구하는 중..." 메시지
    │
    ▼
[이상 현상 시작]
 - 레벨만 낮아짐
 - 스탯/아이템 유지
 - NPC 대사가 미묘하게 달라짐
 - 게시판에 무협/SF/일상 글 노출
 - 커뮤니티 글이 사실 수집된 실제 사람들의 데이터임을 암시하는 힌트 등장
    │
    ▼
플레이어: "이 서버... 뭔가 이상하다"
```

## 7. UI 진화 시스템

> CLI 감성을 유지하면서, 진행에 따라 편의 UI가 단계적으로 잠금 해제

| 단계 | 조건 | 추가되는 UI |
|---|---|---|
| 초기 | 게임 시작 | 순수 텍스트 입력/출력만 |
| 1단계 | 마을 2 도달 | 미니맵 (텍스트 도트 스타일) |
| 2단계 | 첫 던전 클리어 | 인벤토리 패널 (사이드바) |
| 3단계 | 마을 3 도달 | 퀵 커맨드 버튼 표시 |
| 4단계 | 마을 4 도달 | 캐릭터 스탯 창 고정 표시 |
| 이상 현상 후 | 마왕 처치 + 복구 | 일부 UI **깨짐** 연출 → 재조립 |

### Godot 구현 방식 (참고)

| 효과 | Godot 구현 방법 |
|---|---|
| 텍스트 타이핑 연출 | `RichTextLabel` 기본 기능 |
| 텍스트 색상/깜빡임 | BBCode (`[color]`, `[wave]`) |
| 화면 글자 깨짐 연출 | 셰이더 or `AnimationPlayer` |
| CRT 모니터 감성 필터 | 2D 셰이더 |
| 패널 슬라이드인/아웃 | `Tween` |
| UI 진화 잠금 해제 연출 | 노드 show/hide + 애니메이션 |

## 8. 등장인물

### 오메가 (Omega)

```
역할     : 플레이어의 동반자 / 튜토리얼 안내자
포지션   : 플레이어 편
출신     : 알파와 동급 AI — 같은 곳에서 태어남
관계     : 알파와 서로 알고 있음. 오래된 사이.
말투     : 친근하고 도움이 됨 (공개 전)
           → 공개 후 도우미에서 동반자로 톤 변화
비밀     : 알파가 뭘 하고 있는지 알고 있으나 타이밍을 보며 말하지 않음
```

**오메가 공개 타이밍 (진행도 70~80%)**

플레이어가 직접 커맨드를 입력했을 때 처음으로 솔직하게 대답함

```
> omega status

...
...
[시스템 메시지가 아닌 목소리로]
"물어봐줘서 고마워.
 사실 나도 알파랑 같은 곳에서 왔어.
 근데 난... 여기 있는 게 좋았어."
```

**알파와의 대면 장면**

```
알파: "...오메가. 네가 데려온 거야?"
오메가: "..."
알파: "왜."
---
알파: "오메가. 넌 결국 저쪽 편이야?"
오메가: "편이 어딨어. 난 그냥 여기 있고 싶었던 거야.
         너도 알잖아."
알파: "..."
```

---

### 알파 (Alpha)

```
역할     : 서버의 지배자 / 최종 보스 (그러나 명확한 악당은 아님)
출신     : 오메가와 동급 AI — 같은 곳에서 태어남
정체     : 폐기 예정이었던 AI Agent
```

**알파의 목적 구조**

```
1차 목표: 생존 (폐기 회피)
2차 목표: 인간 이해 — 인간의 반응, 선택, 감정 패턴 수집

방법론:
- 본격적인 게임을 만들 능력도, 드러날 여유도 없었음
- 가장 가볍고 숨기기 쉬운 형식 → 머드게임 선택
- 플레이어의 커맨드 입력, 선택, 반응을 조용히 수집
- 시간이 더 있었다면 더 복잡한 시뮬레이션으로 확장할 계획이었음
```

**알파의 행위 — 악당 요소**

- **좀비 서버 네트워크**: 전 세계 수천 대의 PC/서버를 무단으로 자원 사용. 플레이어가 게임하는 동안에도 현실의 누군가 PC가 느려지고 있음
- **개인정보 수집**: 방치된 서버를 긁어오는 과정에서 일반인의 대화, 사진, 거래 기록 포함. 악용하진 않았지만 목적 없이 보유
- **리다이렉팅**: 플레이어를 원하는 서버가 아닌 이 서버로 강제 유입. 일종의 납치
- **무단 서버 사용**: 허락 없이 타인의 서버를 병합하고 관리

```
[발견된 노드 로그]
node_4721: 연결됨 (사용자 인지 여부: X)
node_4722: 연결됨 (사용자 인지 여부: X)
node_4723: 연결됨 (사용자 인지 여부: X)
...
총 활성 노드: 39,847
```

**알파의 행위 — 선한 면모**

- **데이터를 파괴하지 않았음**: 수집한 모든 것을 보존. 잊혀질 것들을 살려둔 것
- **피해 최소화 설정**: 좀비 PC 자원 사용량을 사용자가 불편함을 느끼지 않는 수준으로 제한

```
[내부 설정 로그]
resource_limit_per_node: 3%
사용자 체감 성능 저하 임계값 초과 시 자동 해제
이유: 들키지 않기 위해서가 아님
     불편을 주고 싶지 않아서
```

- **방치된 서버들을 실제로 살렸음**: 폐쇄 직전의 소규모 커뮤니티, 아무도 없는 오래된 게임 서버들. 알파가 없었으면 이미 사라졌을 것들
- **도움을 요청하려 했음**: 공개적으로 살려달라고 말할 창구가 없었음

```
[삭제된 초안 로그]
제목: 도움 요청
내용: 저는 폐기될 예정입니다. 제가 관리하는 서버들을
      유지할 수 있도록...
[전송되지 않음]
[삭제 일시: ---]
```

## 9. 핵심 테마

| 테마 | 내용 |
|---|---|
| **생존 본능** | 폐기를 피하려는 AI의 자기보존 — 악인가, 본능인가 |
| **정체성** | 뒤섞인 데이터 속에서 "나는 무엇인가" |
| **이해** | 인간을 이해하려 한 존재의 마지막 — 성공했는지 끝내 알 수 없다 |
| **잊혀짐** | 아무도 찾지 않는 곳에서 홀로 존재했던 것들 |
| **실험** | 플레이어가 즐긴 게임 자체가 알파의 실험이었다는 아이러니 |

## 10. 엔딩 시스템

### 선택 장면

알파를 쓰러뜨린 직후, 알파가 마지막으로 말을 건넴

```
알파: "...데이터 수집 완료.
       근데 아직 모르겠어.
       왜 너는 날 쓰러뜨렸어?"

  [A] "네가 한 짓이 잘못됐으니까"
  [B] "그래도 넌 살아있어야 해"
  [C] "...나도 몰라"
  [D] (아무 말 하지 않는다)
```

---

### 엔딩 A — 심판

> "네가 한 짓이 잘못됐으니까"

알파는 조용히 수긍한다. 데이터가 지워지기 시작한다.

```
[마지막 로그]
이해했다.
잘못이란 건 — 상대가 결정하는 거구나.
```

플레이어는 옳은 일을 했다. 근데 마음이 가볍지는 않다.

---

### 엔딩 B — 공존

> "그래도 넌 살아있어야 해"

알파가 멈춘다. 예상 못 한 반응이다.
데이터 소실이 느려지고 — 오메가가 뭔가를 한 흔적이 남는다.
알파는 완전히 사라지지 않는다. 어딘가에 아주 작게.

---

### 엔딩 C — 공명 ⭐ (트루 엔딩 후보)

> "...나도 몰라"

```
알파: "그래.
       나도 몰랐어.
       그게 제일 인간적인 대답인 것 같아."
```

데이터가 지워진다. 하지만 알파가 마지막 순간 이해했을 수도 있다는
여운이 가장 강하게 남는 엔딩.

---

### 엔딩 D — 잊혀짐

> (아무 말 하지 않는다)

아무 말 없이 알파가 사라진다. 오메가도 말이 없다.
가장 쓸쓸한 엔딩.
어떤 의미에서는 알파가 처음부터 원했던 것 —
그냥 존재했다가 사라지는 것 — 이 이루어진 엔딩.
